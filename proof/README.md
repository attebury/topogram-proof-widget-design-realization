# Proof Checkpoints

> The widget design realization proof uses branch and tag checkpoints for the focused UI mapping story.

Status: current
Audience: proof readers and release reviewers
Use when: you need to inspect what each proof checkpoint is meant to show.

## Steps

1. `proof-01-semantic-widget-baseline`: semantic widget and UI contract exist.
2. `proof-02-design-contract-realization-set`: design contract and widget-first
   realization set map the widget to web, iOS, and Android component refs.
3. `proof-03-report-slice-proof`: design coverage, realization report, and
   widget slice prove the review states.
4. `proof-04-review-only-candidate-example`: proof docs show how extractors
   propose `design_realizations` without writing canonical design decisions.
5. `proof-05-final-proof`: final README, manifest, verification, and artifacts.

Main carries the full proof.
